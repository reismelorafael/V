────────────────────────────────────────────────────────────────────────────
PADRÕES DE VULNERABILIDADE PREVALENTES
────────────────────────────────────────────────────────────────────────────

Padrão 1: Resource Acquisition Without Release (8 bugs)
Descrição: Streams, readers, cursors, bitmaps alocados mas não fechados se
exceção ocorre. Causa gradual resource exhaustion.
Impacto: OOM, file descriptor exhaustion, database locks.

Padrão 2: Static Reference to Instance (6 bugs)
Descrição: Static fields mantêm referências a Activity, Handler, Listener
instances causando memory leak grave. Particularmente crítico em rotation.
Impacto: Activity nunca é garbage collected, OOM após 10-15 rotações.

Padrão 3: Unremoved Callback/Listener (7 bugs)
Descrição: Listeners registrados em onCreate não removidos em onDestroy.
Referência em static collection causa leak.
Impacto: Memory leak progressivo, ANR em uso prolongado.

Padrão 4: Handler Message Queue Leak (4 bugs)
Descrição: postDelayed ou post sem correspondente removeCallbacks.
Mensagens aguardando na queue retêm Activity.
Impacto: Activity leak, pode ser 5-30 segundos por mensagem.

Padrão 5: Unsafe File Operation (4 bugs)
Descrição: Path não normalizado, symlinks não validados, traversal possível.
Impacto: Acesso a arquivos fora da pasta app, privilege escalation.

Padrão 6: Trust Untrusted Input (6 bugs)
Descrição: Intent extras, JSON, properties files parseados sem validação.
Impacto: Crash, deserialization RCE, injection attack.
